# Алгоритм работу

1. Сихронизируем fork (github)
2. Обновляем локальный репозиторий (git pull, git fetch), подтягиваем изменения из удаленного репозитория. (vs code)
3. Смотрим номер задачи, создаем ветку по номеру задачи (git branch название_ветки(task[номер задачи]))
4. Переходим на ветку с задачей (git checkout название_ветки)
5. Решаем задачу, коммитим изменения (git status, git add ., git commit -m "сделал номер_задачи")
6. Переходим на ветку main (git checkout main)
7. Делаем слияние веток (git merge название_ветки_с_номером_задачи)
8. Удаляем ветку, которую создавали (git branch -d название_ветки)
9. Пушим изменения в удаленный репозиторий (git push)
10. Делаем pull request (github)

Повторяем алгоритм.
